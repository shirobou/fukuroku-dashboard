1. 朝のブリーフィング 9:00〜

トリガー

Claudeに「おはよう」と伝えるだけで自動実行されます。

自動処理の内容

  • Google Calendarから今日の予定を取得
  • Xserverメールの未読メール（過去3日分）を取得
  • 自動通知メール（noreply、Amazon等）は自動除外
  • 結果をDAILY.mdの先頭に記録
  • XserverウェブメールをBraveで自動オープン

出力イメージ

DAILY.mdに追記される内容
## 朝のブリーフィング (2026年3月13日（木） 9:00)

### 今日の予定
- **10:00-11:00** 田中さん面接 @ 事務所
- **14:00-14:30** オンラインMTG

### 未読メール (3件)
- **ジョブメドレー** — 新規応募のお知らせ
  3/13 08:30 | 山田花子様よりご応募が...
- **南先生** — 社会保険手続きの件
  3/12 17:00 | お世話になっております...

2. メール確認・対応 9:30〜

よくある対応パターン

  • 応募メール → 「○○さん応募」でClaudeが面接調整メール生成
  • 面接日程の返信 → カレンダーに登録
  • 問い合わせ → 回答メールを作成
  • 社労士からの連絡 → 内容確認・対応
メール文面はClaudeが自動生成しますが、送信前に必ず確認してください。承認後に送信されます。

3. メール常駐監視 常時

動作

  • 15分間隔で自動チェック
  • 自動通知メール（noreply等）は除外
  • 重要メールが来たら通知メールを送信
  • 同じメールの重複通知は3日間防止

コマンド

実行例
npm run mail-watch        # 常駐監視（15分間隔で継続）
npm run mail-watch:once   # 1回だけチェック

通知メールの例

件名
[メール通知] 2件の重要メール

3件の重要メールがあります（確認: 14:30）

1. ジョブメドレー — 新規応募のお知らせ
   3/13 14:15 | 山田花子様よりご応募が...

4. 書類作成・タスク処理 随時

Claudeへの依頼でできること

  • 「○○さん採用」→ 契約書類一式を自動生成
  • 「経費処理して」→ レシートOCR＆シート転記
  • 「レセプト処理」→ iBowでの半自動処理
  • 「ブログ書いて」→ WordPress記事を自動生成
  • 「社労士に連絡」→ 報告メール自動作成
トリガーワードの一覧は docs/workflows.md に定義されています。自然な日本語で依頼すればOKです。

5. 業務記録・引き継ぎ 17:00〜

DAILY.mdの構成

  • 本日の工程表：時間帯別の予定
  • タスク一覧：優先度順のチェックリスト
  • 実行した業務記録：完了したタスク
  • 気づき・改善案：業務改善のメモ
  • 明日への引き継ぎ：翌日の担当者への申し送り
