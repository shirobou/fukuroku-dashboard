自己負担金管理フロー

利用者の自己負担金をiBowからGoogle Sheetsで月次管理

初期セットアップ（初回のみ）

1 管理シートのセットアップ 自動

Google Sheetsにヘッダー・書式・条件付き書式・プルダウンを自動設定。フィルタービューも作成。

npm run copay:setup -- <シートID>
月次運用

2 iBowから利用者を同期 自動

iBowの利用者一覧から新規利用者のみ自動追加。既存データは一切上書きしない。

npm run copay:sync

3 請求書・領収書の管理 手動

毎月の請求書送付・領収書送付・入金確認をシート上で記録。ステータスは記号で管理。

4 アプラス口座振替の管理 手動

新規利用者の口座振替依頼書の送付状況を管理。引落日は毎月27日。

5 フィルタービューで確認 手動

「現行のみ」「全員表示」の2つのビューで利用者を絞り込み。終了者は赤背景で表示。

ふくろく訪問看護ステーション ／ 最終更新: 2026年3月13日

1. 管理シートのセットアップ

コマンド

実行例
npm run copay:setup -- 1oW375J9jvUpfNtFuvezhhCYnsQJxHxa1LzUGQQX6jWI
npm run copay:setup -- 1oW375J9jvUpfNtFuvezhhCYnsQJxHxa1LzUGQQX6jWI 2027

シートの列構成（A〜K列）

  • A：氏名 ／ B：ふりがな ／ C：施設名/居宅
  • D：保険種別（医療保険/介護保険/生活保護/自立支援）
  • E：訪問開始日 ／ F：訪問終了日
  • G：アプラス送付 ／ H：振替方法
  • I：初回引落日 ／ J：状態 ／ K：備考

月次列（L〜AU列・12ヶ月分）

各月に3列ずつ：請求書（紫）・領収書（青）・入金（緑）

2. iBowから利用者を同期

処理の流れ

  • iBowにログイン → 利用者一覧を取得
  • 氏名・ふりがな・保険種別・施設名を抽出
  • 新規利用者のみ追加（既存データは絶対に上書きしない）
  • デフォルト値：振替方法=口座振替、状態=現行

保険種別の変換ルール

  • iBow「生活保護/生保」→ 生活保護
  • iBow「自立支援/公費」→ 自立支援
  • iBow「医療/健保/国保/後期」→ 医療保険
安全設計：既存の利用者データ（手入力した訪問日・ステータス等）は一切上書きされません。新規追加のみ。

3. 請求書・領収書の管理

ステータス記号

◎ 完了
△ 対応中
✕ 未対応
– 不要

月次管理（各月3列）

  • 請求書（紫背景）：送付済み→◎、準備中→△、未送付→×
  • 領収書（青背景）：送付済み→◎、準備中→△、未送付→×
  • 入金（緑背景）：入金済み→◎、引落待ち→△、未入金→×
条件付き書式：記号を入れるだけで自動で色が変わります。「終了」状態の利用者は行全体が赤背景になります。

4. アプラス口座振替の管理

G列「アプラス送付」のステータス

  • ◎：送付済み・登録完了
  • △：送付済み・登録待ち
  • ×：未送付

口座振替スケジュール

  • 引落日：毎月27日
  • 初回引落：2ヶ月分まとめて
  • 例：1月開始 → 3/27に1月+2月分、4/27に3月分〜

H列「振替方法」

  • 口座振替：アプラス経由（通常）
  • 直接振込：引落不能時や特殊ケース

5. フィルタービュー

用意されているビュー

  • 「現行のみ（終了を非表示）」 — 現在の利用者だけ表示
  • 「全員表示」 — 終了者も含めて全員表示
行1〜3（タイトル・ヘッダー）とA列（氏名）は固定されているので、スクロールしても常に見えます。
