ブログ（WordPress）

自動処理の流れ

  • トピックプールから自動選択（またはカスタムテーマ指定）
  • GPT-4oで2,500〜3,500文字のHTML記事を生成
  • DALL-E 3でアイキャッチ画像（1792x1024px）を生成
  • WordPressに自動投稿（下書き or 公開）

記事の構成ルール

  • SEO最適化タイトル（32文字以内、キーワード先頭）
  • メタディスクリプション（120文字以内）
  • H2見出し 3〜5本
  • 段落は1〜2文まで
  • 料金は具体的な金額を書かない（「お気軽にお問い合わせください」）

コマンド

実行例
npm run blog                    # トピック自動選択
npm run blog -- --date 2026-03-14  # 日付指定
npm run blog:topics             # トピック管理
npm run blog:list               # 下書き一覧
自動投稿スケジュール：毎週金曜9:00。VBS非表示実行で自動化済み。

Instagram カルーセル

スライド構成

  • 表紙：タイトル + カテゴリラベル
  • 本文スライド（5〜8枚）：番号 + タイトル（7文字以内）+ 本文（12文字x2〜3行）
  • 最終スライド：CTA（電話ボタン + 保存を促す文言）
  • 全スライドにページインジケーター + ウォーターマーク(@fukuroku_houkan)

プリセットテーマ（3種類）

  • テーマ1：「訪問看護のキホン」（緑系・8枚）
  • テーマ2：「介護保険で受けられるサービス」（青系・5枚）
  • テーマ3：「ご家族が疲れないための5つの工夫」（暖色系・5枚）

コマンド

実行例
npm run insta:carousel             # テーマ1
npm run insta:carousel -- --theme 2  # テーマ2
npm run insta:carousel -- --all      # 全テーマ一括
画像サイズ：1080x1350px（4:5 PNG）。output/ に保存されます。

X（旧Twitter）投稿

投稿ルール

  • 120〜140文字（120文字未満はNG）
  • ハッシュタグは最大2個（文字数に含む）
  • 親しみやすい自然なトーン（AI感を出さない）
  • 具体例や「あるある」を含める
  • コンテンツパックでは3つの異なる切り口を生成

コマンド

実行例
/x                        # 3案生成 → 承認 → 投稿
npm run x:post "本文"     # 直接投稿
npm run x:queue add "本文" "2026-03-14 09:00"  # 予約
npm run x:queue:list      # キュー確認
npm run x:schedule        # 予約済みを実行

コンテンツパック一括生成

1テーマ → 3チャネル自動展開

  • Instagram：カルーセル画像セット + キャプション
  • X：3つの異なる切り口の投稿案
  • ブログ：SEO記事 + アイキャッチ画像

コマンドオプション

実行例
npm run content-pack -- "訪問看護の選び方"     # 全チャネル
npm run content-pack -- --instagram-only      # Instagramのみ
npm run content-pack -- --blog-only           # ブログのみ
npm run content-pack -- --x-only              # Xのみ
npm run content-pack -- --no-blog             # ブログ以外
npm run content-pack -- --publish             # ブログを公開状態で
