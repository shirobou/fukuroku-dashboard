準備（月末前〜月末）

1 従業員への勤怠確認依頼 手動 月末前

毎月月末に従業員へ当月の勤怠確認を依頼します。各自がKOTのマイメニューから打刻確認を行い、「勤怠確認済みとする」をクリックしてもらいます。

KOT > マイメニュー > 打刻確認 > 勤怠確認済みとする

2 関連資料の印刷 手動 月末

共有フォルダから「勤務体制一覧表」「出勤予定表」「出勤実績表」の3点を印刷し、手元に準備します。

KOT処理（月末〜翌月1日）

3 打刻入力（自動）+ 修正（手動） 自動 手動 月末

スマホ操作が苦手な6名（志摩・松生・宮地・中尾・木村・田中）はスクリプトで一括打刻。打刻抜け・欠勤・遅刻早退などの修正は引き続き手動で対応します。

npm run kingoftime:punch -- --month=YYYY-MM

4 振替休日・振替出勤の処理 手動 月末

シフト振替がある場合、予定と実績の差異を残す形でKOTに入力します。振休の日と振替出勤の日で入力方法が異なります。

5 最終確認 手動 翌月1日

「スケジュール管理」画面を再度確認し、予定と実績が完全に一致しているかチェックします。

出力・共有（翌月1日〜初旬）

6 タイムカードPDF出力・保存・ファイリング 手動 翌月1日

タイムカードをPDFで出力し、所定フォルダへ保存。印刷したタイムカードをファイリングします。

KOT > エクスポート / インポート > タイムカード［PDF］

7 南先生（社労士）へ月次データ共有 手動 翌月初旬

タイムカードPDFに加え、経費の立替分や自家用車通勤者のガソリン代をまとめて南先生にメールで共有します。

翌月準備

8 翌月スケジュール登録（自動） 自動 翌月初旬

共有フォルダのExcelに翌月予定を入力 → スクリプトで訪問看護部門10名分を一括登録します。

1. 従業員への勤怠確認依頼 月末前

タイミング

毎月月末（勤怠締め作業の前）に、全従業員へ当月の勤怠確認を依頼します。

従業員に依頼する操作手順

  • KOTにログイン
  • マイメニュー を開く
  • 打刻確認 を開く
  • 当月の打刻内容を確認し、問題がなければ 「勤怠確認済みとする」 をクリック

確認のポイント

  • 打刻漏れや時刻の誤りがないか各自で確認してもらう
  • 修正が必要な場合は事務へ連絡してもらう
  • 全員の確認が完了してから次のステップ（資料印刷・KOT修正）に進む
従業員本人に勤怠を確認してもらうことで、打刻抜けや誤りの早期発見につながります。全員の「勤怠確認済み」が揃ってから締め作業を開始してください。

2. 関連資料の印刷 月末

印刷する資料（3点）

  • 従業者の勤務の体制及び勤務形態一覧表
  • 出勤予定表
  • 出勤実績表

保管場所

共有フォルダから取得します。印刷後、手元に並べてKOTの画面と照合できる状態にしておきます。

この3点を見比べながらKOTの入力を行うので、必ず先に印刷してから次のステップに進んでください。

3. 打刻入力（自動）+ 修正（手動） 月末

① 自動打刻（6名・スクリプト） 自動

スマホ操作が苦手な以下6名の打刻は、スケジュール時刻 ±5分のランダムで一括入力します。既存打刻ありの日はスキップされるので何度でも安全に実行可能です。

対象6名志摩 早苗、松生 仁美、宮地 あけみ、中尾 梨加、木村 恵美、田中 佐和子
出勤打刻予定9:00 → 8:55〜9:00 ランダム
退勤打刻予定17:30 → 17:30〜17:35 ランダム
事前確認（推奨）npm run kingoftime:punch:dry -- --month=YYYY-MM
本番実行npm run kingoftime:punch -- --month=YYYY-MM
引数 --month を省略するとプロンプトで月を聞かれます。Claudeに「今月分の打刻入れて」と頼むだけでもOK。

② 手動修正（その他全員） 手動

その他の従業員はスマホで自打刻済み。打刻抜け・欠勤・遅刻早退などの修正のみ手動で対応します。

  • KOTにログイン（https://s4.ta.kingoftime.jp/admin）
  • 全メニュー > スケジュール管理 を開く
  • 対象者の タイムカード を開く
  • 印刷資料と照合しながら、打刻抜け・欠勤などを入力

よくある修正パターン

  • 打刻忘れ：出勤 or 退勤の打刻が片方だけ → 実際の時刻を手入力
  • 欠勤：予定は出勤だが休んだ → 休暇区分を設定
  • 遅刻・早退：打刻時刻が予定と異なる → 実績をそのまま残す

4. 振替休日・振替出勤の処理 月末

考え方

予定していたシフトを振り替えた場合（例：16日の出勤を休みにし、28日の休みを出勤に変更）は、予定と実績の差異を残すため、以下の手順で入力します。

振替休日の日（例：16日 ― 予定は出勤、実際は休み）

項目設定値
パターン通常勤務（9:00〜17:30）のまま
休暇区分振休（または振替休日）を選択
取得単位全日休暇を選択
保存対象日にチェック → 保存

振替出勤の日（例：28日 ― 予定は休み、実際は出勤）

項目設定値
パターン通常勤務（9:00〜17:30）を選択
勤務日種別平日を選択
備考16日の振替出勤と入力
保存対象日にチェック → 保存
振替出勤の備考欄には、必ずどの日の振替かを記入してください。後から確認する際に追跡しやすくなります。

5. 最終確認 翌月1日

チェックポイント

  • 「スケジュール管理」画面で全従業員の予定と実績を再確認
  • 印刷した資料（勤務体制一覧表・出勤予定表・出勤実績表）との整合性を確認
  • 振替休日・振替出勤の入力が正しいか確認
  • 打刻抜けや未処理の日がないかチェック
ここで修正漏れがあるとタイムカードPDFに反映されてしまいます。出力前の最終チェックなので丁寧に確認してください。

6. タイムカードPDF出力・保存・ファイリング 翌月1日

PDF出力手順

  • 全メニュー > エクスポート / インポート を開く
  • データ出力項目から タイムカード［PDF］ を選択
  • 出力対象年月を設定
  • 出力レイアウト（横フォーマットなど）を選択
  • データ出力をクリック

保存先

ファイル保存先パス
\\192.168.0.60\share\出勤簿\（年度）（職員打刻表）

※年度は対象年度に合わせて変更してください（例：2026年度（職員打刻表））

ファイリング

  • 印刷したタイムカードをファイリング（紙保管）

7. 南先生（社労士）へ月次データ共有 翌月初旬

共有するデータ

以下の3点をまとめて南先生にメールで送付します。

  • タイムカードPDF ― KOTから出力したもの（備考欄に振替・欠勤理由を記載済み）
  • 経費の立替分 ― 従業員が立て替えた経費の明細
  • ガソリン代 ― 自家用車通勤者の走行距離に基づくガソリン代

送付先

南先生メールアドレス
minami300.28.86@gmail.com

メール送信方法

  • Claudeに「南先生にタイムカードと経費を送って」と依頼
  • 文面を確認してから送信（自動では送信されません）
  • 送信済みフォルダ（INBOX.Sent）に自動保存されます
経費立替やガソリン代は、経費精算フローで集計したデータを活用します。タイムカードと一緒にまとめて送ることで、南先生側での給与計算がスムーズになります。

8. 翌月スケジュール登録（自動） 翌月初旬

① Excelに翌月予定を入力（事前作業）

共有フォルダの勤務表Excelを開き、対象月のシートに各人の予定を入力します。「日」=通常出勤、「公」=公休、空欄=該当なし。

Excelファイルパス
\\192.168.0.60\share\出勤簿\2026年勤務表（予定・実績）\勤務表（2026年度）.xlsx

※対象シート名は YYYYMM（予定） 形式（例: 202606（予定））

② スクリプトで一括登録 自動

訪問看護部門10名（正社員2名＋パート8名）の予定をKOTに一括登録します。「通常」「公休（法定/法定外を週単位で自動判定）」も自動処理。

事前確認（推奨）npm run kingoftime:schedule:dry -- --month=YYYY-MM
本番実行npm run kingoftime:schedule -- --month=YYYY-MM
登録内容パート: 通常出勤のみ／正社員: 通常出勤＋公休（法定休日/法定外休日を自動振り分け）
既存上書きOK（チェックON＝上書き、OFF＝既存維持）
Excelを更新しておけば、Claudeに「来月分のスケジュール入れて」と頼むだけでOK。法定休日/法定外休日は「週内で最後の公休=法定休日、それ以外=法定外」のルールで自動分類します。

補足：従業員の表示順を変えたい場合

  • 全メニュー > 従業員 > 従業員設定 を開く
  • 表示条件の指定 で部門（訪問看護部門など）を選択し「表示」
  • 表示順設定 ボタンから並び順を変更
翌月スケジュールの登録まで完了したら、タイムカード処理・勤怠締め作業はすべて完了です。お疲れさまでした！
